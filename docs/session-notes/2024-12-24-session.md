## Completed Work

### 1. Email Ingestion Error Handling (First Round)

**Commits:** `11f3667`

Added proper error categorization and auto-deactivation for email ingestion configs:

- Created `PermanentEmailError` and `TransientEmailError` exception classes in `shared/exceptions/email.py`
- Permanent errors (folder doesn't exist, auth failed) → deactivate immediately
- Transient errors (connection issues) → retry up to 3 times, then deactivate
- Added `deactivate_config_with_error()` to EmailService
- Fixed IMAP folder selection for folder names with spaces (quoting)
- Clear error fields when reactivating a config

### 2. Removed Unused HTC Integration Router

**Commit:** `b167e3e`

Deleted `server/src/api/routers/htc_integration.py` - the `GET /htc/orders/{order_number}` endpoint was unused. The order_management router fetches HTC fields internally via the service.

### 3. Pending Update Field Comparison Fixes

**Commit:** `5b32782`

**Dims comparison issue:** Dims were being compared as JSON strings, causing false positives due to:
- Integer vs float differences (`10` vs `10.0`)
- Extra `dim_weight` field in HTC data
- Key ordering differences

**Fix:** Added `_dims_are_equal()` helper in `output_processing/service.py` that:
- Parses both values to Python objects
- Normalizes numeric values
- Ignores `dim_weight` (calculated field)
- Compares as sets of tuples

**Address comparison:** Added debug logging with `[ADDRESS COMPARE]` and `[FIND_ADDRESS_ID]` prefixes to help diagnose potential false positives.

### 4. Dim Builder Module Reorder

**Commit:** `3cf39cb`

Changed input order from `(height, length, width, qty, weight)` to `(count, length, width, height, weight)` for better usability.

### 5. Email Reconnection Fix (Second Round)

**Commit:** `1dcb810`

The first round of error handling still had a critical bug: when the IMAP connection died (network issues), the system would count errors toward deactivation and disable the config after 3 failures.

**Root cause:** When keepalive NOOP failed, it logged a warning but didn't clear `self._imap`, so `_ensure_connected()` thought the connection was still valid.

**Fixes:**
- Keepalive now sets `self._imap = None` when NOOP fails
- Added `_is_connection_error()` to detect socket/SSL errors
- Added `_handle_connection_error()` to clean up dead connections
- Email operations (`get_emails_since_uid`, `get_highest_uid`, `get_attachments`) now catch connection errors, clear the connection, and raise `TransientEmailError`
- **Key change:** `TransientEmailError` no longer counts toward deactivation - just logs and retries

**New behavior:** Connection errors trigger reconnection on next poll cycle. Deactivation only happens for permanent errors.

### 6. AuthContext Refactor

**Commit:** `f45a478`

Replaced hardcoded `fetch()` calls with centralized `apiClient` in AuthContext. Added `envDir` to vite.config.ts for .env loading.
