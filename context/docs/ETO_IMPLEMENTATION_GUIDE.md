## Architecture Patterns

### Layer Separation

```
Database (SQLAlchemy Models)
    ↕ (Repository converts)
Domain Types (Dataclasses)
    ↕ (Service orchestrates)
Service Layer (Business Logic)
    ↕ (API routes or Worker)
API / Worker
```

### Key Principles

1. **Domain Types** = Pure dataclasses, no business logic
2. **Repositories** = Database CRUD only, no business logic
3. **Services** = Business logic, orchestration, multiple repository calls
4. **Models use Enums** → **Domain types use Literal strings**

## 1. Domain Types Pattern

### Location
`server-new/src/shared/types/{table_name}.py`

### Pattern (Based on `eto_runs.py`)

```python
"""
{TableName} Domain Types
Dataclasses representing {table_name} table and related operations
"""
from dataclasses import dataclass
from datetime import datetime
from typing import Literal, Optional

# =========================
# Type Aliases for Enums
# =========================

# Match enums from models.py exactly
{EnumName} = Literal["value1", "value2", "value3"]


# =========================
# Domain Types
# =========================

@dataclass
class {TableName}Create:
    """
    Data required to create a new record.
    Only include fields that are NOT auto-generated by database.
    """
    required_field_1: type
    optional_field_1: Optional[type] = None


@dataclass
class {TableName}Update:
    """
    Data for updating a record.
    All fields optional - only provided fields will be updated.
    """
    field_1: Optional[type] = None
    field_2: Optional[type] = None


@dataclass
class {TableName}:
    """
    Complete record as stored in the database.
    Represents the {table_name} table exactly.
    Must include ALL columns from the database model.
    """
    id: int
    # ... all fields from model
    created_at: datetime
    updated_at: datetime
```

### Steps to Create Domain Types

1. **Open the database model** in `models.py`
2. **Identify all columns** (use `Mapped[type]` annotations)
3. **Note enums** - Convert to Literal type aliases
4. **Create `{Name}Create`**:
   - Include only fields user provides
   - Exclude: `id`, `created_at`, `updated_at`, any auto-generated fields
5. **Create `{Name}Update`**:
   - All fields optional (for partial updates)
   - Exclude: `id`, `created_at`, `updated_at` (never update these)
6. **Create `{Name}`**:
   - Include EVERY field from the model
   - Match types exactly (but use Literal strings for enums)

### Example: Template Matching Types

**From `EtoRunTemplateMatchingModel` in models.py:**
```python
# Model has:
id: Mapped[int]
eto_run_id: Mapped[int]
status: Mapped[EtoStepStatus]  # Enum: processing, success, failure
matched_template_version_id: Mapped[Optional[int]]
started_at: Mapped[Optional[datetime]]
completed_at: Mapped[Optional[datetime]]
created_at: Mapped[datetime]
updated_at: Mapped[datetime]
```

**Domain types should be:**
```python

@dataclass
class EtoRunTemplateMatchingCreate:
    eto_run_id: int
    # status will default to "processing" in repository

@dataclass
class EtoRunTemplateMatchingUpdate:
    status: Optional[EtoStepStatus] = None
    matched_template_version_id: Optional[int] = None
    completed_at: Optional[datetime] = None

@dataclass
class EtoRunTemplateMatching:
    id: int
    eto_run_id: int
    status: EtoStepStatus  # String literal, not enum
    matched_template_version_id: Optional[int]
    started_at: Optional[datetime]
    completed_at: Optional[datetime]
    created_at: datetime
    updated_at: datetime
```

## 2. Repository Pattern

### Location
`server-new/src/shared/database/repositories/{table_name}.py`

### Pattern (Based on `eto_run.py`)

```python
"""
{TableName} Repository
Repository for {table_name} table with CRUD operations
"""
import logging
from typing import Type, Optional, List

from shared.database.repositories.base import BaseRepository
from shared.database.models import {TableName}Model
from shared.types.{table_name} import (
    {TableName},
    {TableName}Create,
    {TableName}Update,
)

logger = logging.getLogger(__name__)


class {TableName}Repository(BaseRepository[{TableName}Model]):
    """
    Repository for {table_name} CRUD operations.

    Handles:
    - Basic CRUD operations
    - Conversion between ORM models and domain dataclasses
    - Common query operations
    """

    @property
    def model_class(self) -> Type[{TableName}Model]:
        """Return the SQLAlchemy model class this repository manages"""
        return {TableName}Model

    # ========== Conversion Methods ==========

    def _model_to_domain(self, model: {TableName}Model) -> {TableName}:
        """
        Convert ORM model to domain dataclass.
        Handle enum → string conversion here.
        """
        return {TableName}(
            id=model.id,
            # Convert enums: model.status.value
            # Copy datetime as-is: model.created_at
            # Handle optionals: model.field if model.field else None
        )

    # ========== CRUD Operations ==========

    def create(self, data: {TableName}Create) -> {TableName}:
        """Create new record"""
        with self._get_session() as session:
            model = self.model_class(
                field1=data.field1,
                field2=data.field2,
                # timestamps auto-set by server_default
            )
            session.add(model)
            session.flush()
            return self._model_to_domain(model)

    def get_by_id(self, record_id: int) -> Optional[{TableName}]:
        """Get record by ID"""
        with self._get_session() as session:
            model = session.get(self.model_class, record_id)
            if model is None:
                return None
            return self._model_to_domain(model)

    def update(self, record_id: int, data: {TableName}Update) -> Optional[{TableName}]:
        """Update record. Only updates provided fields."""
        with self._get_session() as session:
            model = session.get(self.model_class, record_id)
            if model is None:
                return None

            # Update only provided fields (check is not None)
            if data.field1 is not None:
                model.field1 = data.field1
            # ... repeat for all fields

            session.flush()
            return self._model_to_domain(model)

    # ========== Query Operations ==========
    # Add any specialized queries here
    # Example: get_by_eto_run_id(), get_by_status(), etc.
```

### Key Repository Guidelines

1. **Use `_get_session()` context manager** - Works with BaseRepository pattern
2. **Always use `session.flush()` not `commit()`** - BaseRepository handles commits
3. **Enum Conversion**:
   - Model → Domain: `model.status.value` (enum → string)
   - Domain → Model: `data.status` (SQLAlchemy auto-converts string → enum)
4. **Optional Fields**: Check `if data.field is not None:` before updating
5. **Add query methods** specific to your table needs (e.g., `get_by_eto_run_id()`)

## 4. Worker Pattern

### Location
`server-new/src/services/eto/worker.py`

### Worker Responsibilities

- Run continuously in background
- Poll for `not_started` runs
- Process runs sequentially (or in parallel if designed for it)
- Handle errors without crashing
- Log progress

### Example Worker Structure

```python
"""
ETO Worker
Background worker that processes ETO runs
"""
import logging
import time
from typing import Optional

from services.eto.eto_processing_service import EtoProcessingService
from shared.database.repositories.eto_run import EtoRunRepository

logger = logging.getLogger(__name__)


class EtoWorker:
    """
    Background worker that continuously processes ETO runs.
    Polls for runs with status = "not_started" and processes them.
    """

    def __init__(
        self,
        eto_processing_service: EtoProcessingService,
        eto_run_repo: EtoRunRepository,
        poll_interval_seconds: int = 5,
    ):
        self.eto_processing_service = eto_processing_service
        self.eto_run_repo = eto_run_repo
        self.poll_interval = poll_interval_seconds
        self._running = False

    def start(self):
        """Start the worker loop"""
        self._running = True
        logger.info("ETO Worker started")

        while self._running:
            try:
                self._process_next_run()
            except Exception as e:
                logger.error(f"Worker error: {e}", exc_info=True)
                # Don't crash - continue processing

            time.sleep(self.poll_interval)

    def stop(self):
        """Stop the worker loop"""
        self._running = False
        logger.info("ETO Worker stopped")

    def _process_next_run(self):
        """Find and process one not_started run"""
        # Find oldest not_started run
        runs = self.eto_run_repo.get_by_status("not_started", limit=1)

        if not runs:
            return  # Nothing to process

        run = runs[0]
        logger.info(f"Processing ETO run {run.id}")

        # Process the run
        success = self.eto_processing_service.process_run(run.id)

        if success:
            logger.info(f"ETO run {run.id} completed successfully")
        else:
            logger.error(f"ETO run {run.id} failed")
```

### Worker Guidelines

1. **Never crash** - Catch all exceptions in main loop
2. **Log everything** - Info for progress, Error for failures
3. **Use polling** - Simple `time.sleep()` between checks
4. **Process one at a time** initially (can parallelize later)
5. **Graceful shutdown** - Set `_running = False` to stop

## Common Pitfalls to Avoid

### 1. Enum Handling
❌ **Wrong**: `return model.status` (returns enum object)
✅ **Right**: `return model.status.value` (returns string)

### 2. Partial Updates
❌ **Wrong**: `model.field = data.field` (sets None even if not provided)
✅ **Right**: `if data.field is not None: model.field = data.field`

### 3. Session Management
❌ **Wrong**: `session.commit()` in repository
✅ **Right**: `session.flush()` - let BaseRepository handle commits

### 4. Datetime Handling
❌ **Wrong**: `datetime.now()` (local timezone)
✅ **Right**: `datetime.utcnow()` (UTC timezone)

### 5. Foreign Key References
❌ **Wrong**: Accessing `model.relation.field` without loading
✅ **Right**: Use explicit joins or check if relation is loaded

## Implementation Order

### Recommended Sequence

1. **Complete all domain types first** (types are needed by repositories)
2. **Complete all repositories** (repos are needed by services)
3. **Build services** (orchestration layer)
4. **Build worker** (background processor)
5. **Test end-to-end** (create run → worker processes it)

### Why This Order?
- Bottom-up: Build foundation before higher layers
- Each layer depends on the one below
- Can test each layer independently
